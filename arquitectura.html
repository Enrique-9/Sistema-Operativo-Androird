1.- Arquitectura, desempeño y tipo de un sistema operativo

Android es un sistema operativo móvil creado por Google y basado en el núcleo de Linux. Está diseñado para dispositivos de pantalla táctil como teléfonos y tabletas. Su principal objetivo es ofrecer una plataforma abierta que permite personalizar, innovar y ser compatible entre distintos dispositivos.

1.1 Codigo abierto:

Su código fuente principal (Android Open Source Project o AOSP) está disponible públicamente. Esto permite a fabricantes y desarrolladores modificar y adaptar el sistema operativo, lo que fomenta la innovación y la creación de una amplia oferta de aplicaciones (muchas gratuitas) y una rápida corrección de errores.

Personalizacion:

Ofrece una gran libertad al usuario y al desarrollador para modificar la interfaz de usuario, usar launchers (lanzadores), widgets y personalizar el comportamiento del sistema.

Diversidad y versatilidad:

Se encuentra en una vasta gama de dispositivos, no solo teléfonos móviles, sino también tabletas, relojes inteligentes, televisores (Android TV), y otros electrodomésticos, en una gran variedad de marcas y precios.

Multitarea real:

Permite que las aplicaciones se ejecuten en segundo plano de manera simultanea, facilitando el cambio rapido entre ellas.

Integracion con Google:

Esta perfectamente integrado con servicios como Google Play Store (su catalogo de aplicaciones), Gmail, Google Maps y Google Drive, proporcionando una experiencia cohesiva.

Adaptabilidad:

Es adaptable a multiples resoluciones de pantalla y soporta una amplia variedad formatos multimedia (audio, video, imagenes) y estandares de conectividad (Wi-Fi, Bluetooth, 4G, 5G, etc.).

1.2 Funciones de los componentes de Android

Las funciones del sistema operativo Android se dividen en dos grandes áreas: funciones de bajo nivel, encargadas de la gestión interna del sistema, y funciones orientadas al usuario y al desarrollo, enfocadas en la interacción y ejecución de aplicaciones.

Funciones de Bajo Nivel (Núcleo Linux)

Android está basado en el núcleo de Linux, el cual es responsable del funcionamiento básico del dispositivo y la gestión de sus recursos.

Gestión de recursos de hardware

Actúa como intermediario entre el software y el hardware del dispositivo, permitiendo el uso de componentes como pantalla, cámara y memoria.

Gestión de memoria y procesos

Administra la memoria RAM y permite la multitarea, facilitando que varias aplicaciones se ejecuten al mismo tiempo.

Seguridad del sistema

Proporciona servicios base de seguridad y modelos de permisos como SELinux, protegiendo el sistema contra accesos no autorizados.

Pila de red y conectividad

Gestiona la comunicación de red, incluyendo Wi-Fi, datos móviles y Bluetooth.

Administración de energía

Optimiza el consumo de batería mediante la gestión eficiente de procesos y recursos del sistema.

Funciones orientadas al usuario y desarrollo

Interfaz y experiencia de usuario

Interfaz gráfica (UI): Proporciona elementos visuales como botones, iconos y menús. Es altamente personalizable.


Sistema de notificaciones: Organiza alertas en un panel central.


Multitarea: Permite cambiar entre aplicaciones abiertas y ver tareas recientes.

Soporte de aplicaciones y servicios

ART: Ejecuta aplicaciones dentro de procesos independientes.

Integración y multimedia

1.3 Clasificación de Android

Basándose en sus características y funciones, el sistema operativo Android puede clasificarse en diferentes categorías que describen su naturaleza y su forma de desarrollo.

Tipo de sistema operativo

Sistema Operativo Móvil

Android está diseñado y optimizado para dispositivos con recursos limitados, como batería, memoria y procesador.

Además, está centrado en la interacción táctil y la conectividad inalámbrica, características propias de smartphones y tablets.

Modelo de código

Sistema de Código Abierto (Open Source)

El núcleo del sistema (AOSP) y muchas de sus capas superiores son de libre acceso, uso y modificación.

Esto permite la colaboración entre desarrolladores y la adaptación del sistema por parte de fabricantes (OEMs), fomentando la innovación.

1.4 Interfaces de Android

Android utiliza principalmente una interfaz gráfica para el usuario común, pero también incluye una interfaz de línea de comandos para tareas técnicas y de desarrollo.

GUI (Graphical User Interface)

Definición

Es una interfaz basada en elementos visuales como iconos, botones, menús y ventanas que permiten una interacción intuitiva con el sistema.

Función en Android

Proporciona la experiencia de uso diaria del dispositivo, incluyendo la pantalla de inicio, aplicaciones y sistema de notificaciones.

Interacción

Se basa en la interfaz táctil (TUI), permitiendo al usuario interactuar mediante gestos y toques en la pantalla.

CLI (Command Line Interface)

Definición

Es una interfaz basada en texto donde el usuario interactúa con el sistema mediante comandos específicos.

Función en Android

Está presente a nivel del núcleo Linux y permite la gestión del sistema, procesos y hardware mediante comandos.

Herramientas de desarrollo

Los desarrolladores utilizan herramientas como ADB (Android Debug Bridge) para instalar aplicaciones, depurar código y acceder al sistema.

Usuarios avanzados

Mediante aplicaciones como Termux o dispositivos rooteados, es posible acceder a la shell de Linux y ejecutar comandos de bajo nivel.

1.5 Principales utilerías del sistema

Android está formado por distintos componentes fundamentales que trabajan en conjunto para garantizar el correcto funcionamiento del sistema. Cada uno cumple funciones específicas dentro de la arquitectura del sistema operativo.

Núcleo de Linux

Es la base del sistema y se encarga de funciones esenciales como la seguridad, la gestión de memoria, la administración de procesos, la comunicación en red y el manejo de controladores.

Además, actúa como intermediario entre el hardware y el software.

Capa de Abstracción de Hardware (HAL)

Permite la comunicación entre Android y los componentes físicos del dispositivo sin modificar las capas superiores, facilitando la compatibilidad con diferentes equipos.

Android Runtime (ART)

Es el entorno encargado de ejecutar las aplicaciones, optimizando el rendimiento mediante técnicas como compilación AOT, JIT y recolección de memoria.

Bibliotecas nativas (C/C++)

Proporcionan funciones básicas utilizadas por el sistema y las aplicaciones, sirviendo como soporte para operaciones internas.

Marco de API de Java

Ofrece a los desarrolladores herramientas y APIs que permiten acceder a las funciones del sistema y crear aplicaciones de manera sencilla.

Servicios del sistema

Incluyen funciones como telefonía, localización, gestión de ventanas, administración de paquetes y proveedores de contenido.

Marco de aplicación

Proporciona la estructura necesaria para desarrollar aplicaciones, incluyendo componentes como actividades, servicios, receptores y proveedores.

Aplicaciones

Son la capa superior del sistema. Incluyen tanto aplicaciones preinstaladas como aquellas descargadas por el usuario.

Estas se ejecutan en un entorno seguro que protege el sistema y los datos del usuario.
