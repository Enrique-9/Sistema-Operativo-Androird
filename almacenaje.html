3.- Almacenaje: Memoria y Archivos

En Android, la memoria y el almacenamiento son fundamentales para el rendimiento del sistema, la multitarea, la seguridad y la administración de recursos. Esta sección explica los tipos de memoria física, la administración de memoria, el sistema de archivos y la asignación de memoria entre procesos.

3.1 Tipos de memorias físicas en Android

Los principales dispositivos que usan este sistema operativo son los dispositivos móviles y tabletas, aunque también existen versiones de este mismo sistema operativo para otros dispositivos como Android TV o Android carplay, los cuales son especialmente desarrollados para las TV y los automóviles; una vez se conocen los dispositivos en los que podemos encontrar este sistema operativo podemos pasar a ver que tipos de memorias físicas se usan en estas unidades.

Memoria RAM (Random Acces Memory)

Este tipo de memoria es muy importante en los dispositivos Android, ya que es la que se encarga principalmente de hacer que la multitarea funcione correctamente, específicamente centrándonos en la versión 14 de Android encontramos un tipo de memoria RAM que emplea la tecnología LPDDR (Low-Power Double Data Rate), con esta tecnología los dispositivos logran tener una gran velocidad y un consumo energético reducido, lo cual es de suma importancia en este tipo de dispositivos con capacidades de expansión muy limitadas.

En esta versión de Android se contaba principalmente con cuatro versiones de esta tecnología, enfocadas en diferentes gamas de dispositivos móviles:

LPDDR4: Esta versión de la tecnología LPDDR es la más básica que se podía encontrar en dispositivos gama baja o baratos.

LPDDR4X: Esta es la versión mejorada del LPDDR4 y a diferencia de su antecesor esta versión cuenta con un mejor balance entre velocidad y consumo energético.

LPDDR5: Esta versión es más moderna y permite una relación de velocidad y consumo energético muy superior a las anteriores.

LPDDR5x: Esta es una versión que optimiza el rendimiento y consumo energético en dispositivos de alto rendimiento.

También es importante hacer mención de que algunos dispositivos con este sistema operativo cuentan con una opción activable de memoria virtual, la cual permite tener una mayor capacidad de almacenamiento, aunque hay que tener en cuenta que este ajuste toma parte de la memoria ROM que es bastante más lenta que la memoria RAM.



Memoria ROM (Read-Only Memory)

Estas siglas hacen referencia al almacenamiento interno de los dispositivos, este tipo de almacenamiento es el que se encarga de guardar el sistema operativo, lo datos del usuario y las aplicaciones, al igual que como se había mencionado en el caso de la RAM, la ROM también es muy importante y debe contar con ciertas características para que el dispositivo y sistema operativo funcionen de forma correcta. Los dispositivos con el sistema operativo Android 14 suelen utilizar memoria ROM del tipo flash la cual a diferencia de las ROMs comunes puede ser reprogramada electrónicamente. Dentro de este tipo de memoria contamos con varias tecnologías, una de las más importantes es UFS (Universal Flash Storage), las principales diferencias entre cada una de las versiones de esta tecnología con respecto a sus velocidades de lectura y escritura. Algo importante que se puede destacar es que hasta esta versión de Android la mayoría de los dispositivos contaban con las opciones de aumentar el almacenamiento mediante una SD externa, tecnología que ha ido desapareciendo con el paso del tiempo.

3.2 Administración de memoria en Android

Android administra la memoria (RAM) a través de una combinación de mecanismos heredados del Kernel de Linux y utilidades propias, centrándose en el aislamiento de procesos y la conservación de recursos mediante la eliminación selectiva de aplicaciones inactivas.

Mecanismos Fundamentales (Kernel de Linux)

Android utiliza el kernel de Linux para manejar las funciones de memoria de bajo nivel:

Paginación (Paging)

Es el mecanismo principal de memoria virtual que permite la multitarea segura.

Aislamiento

Divide la memoria de cada proceso en páginas. La Unidad de Gestión de Memoria o MMU utiliza tablas de páginas para mapear direcciones virtuales a físicas, evitando que una app acceda a la memoria de otra.

Ejecución eficiente

Permite ejecutar aplicaciones aunque solo parte del código esté en RAM, cargando lo necesario dinámicamente.

Uso limitado de Swap

Android evita el uso intensivo de swap para no afectar el rendimiento ni desgastar la memoria. En su lugar, libera RAM cerrando procesos inactivos.

zRAM

Utiliza compresión de datos en RAM para mejorar la eficiencia sin consumir batería adicional.

Utilidades Específicas de Android

El sistema operativo añade capas de gestión por encima del kernel para optimizar la RAM en el entorno móvil.

Android Runtime (ART)

Es la máquina virtual encargada de ejecutar las aplicaciones en Android.

Garbage Collector (GC)

Libera automáticamente memoria de objetos no utilizados, evitando fugas de memoria y optimizando el rendimiento.

Zygote

Proceso que precarga clases del sistema. Las apps se crean como copias (fork), lo que acelera su inicio.

Low Memory Killer (LMK)

Administra la memoria cerrando aplicaciones de baja prioridad cuando la RAM es insuficiente.

3.3 Importancia de la administración de memoria en Android

Optimización de recursos

El administrar bien la memoria en Android es muy importante, ya que generalmente los dispositivos que cuentan con este sistema operativo son móviles y la mayoría de las veces se prioriza esta comodidad de movilidad sobre los recursos, por lo que la administración de la escasa memoria es necesaria para poder operar de forma eficiente en este sistema operativo.



Optimización del consumo de energía

Al igual que en el punto anterior la batería de los dispositivos móviles con el sistema operativo Android es un recurso limitado muy valioso, la correcta administración de memoria ayuda a que este recurso limitado sea utilizado de la forma más eficiente posible, esto se logra mediante varios procesos, por ejemplo, al gestionar y dormir activamente los procesos inactivos que consumen energía sin razón, la administración de memoria reduce el consumo de batería, lo que es vital en un dispositivo móvil.



Multitarea

La multitarea en dispositivos con Android es posible gracias a que la administración de memoria se lleva a cabo de forma estructuradamente correcta, lo que permite la ejecución de varios procesos a la vez, aunque no se cuente con los recursos necesarios.

3.4 Sistema de archivos y almacenamiento

El sistema de archivos de Android está basado en el Kernel de Linux y se caracteriza por una separación lógica estricta entre las áreas de almacenamiento, lo que garantiza la seguridad y privacidad de los datos.

Almacenamiento Interno (Privado)

Es el área de memoria integrada del dispositivo reservada para los datos esenciales del sistema operativo y los datos privados de cada aplicación.

Acceso

Solo la aplicación que creó un archivo puede acceder a él. Es un espacio aislado (sandbox) que garantiza que una aplicación maliciosa no pueda leer los datos sensibles de otra (apps de mensajería, bancarias, etc.).

Eliminación

Cuando desinstalas una aplicación, todos sus datos almacenados internamente se eliminan automáticamente.

Uso típico

Almacenar bases de datos SQLite, archivos de configuración (preferences) o caché temporal.

Sistemas de Archivos (Físicos)

Android, basado en Linux, soporta varios sistemas de archivos, pero dos son los más relevantes para las particiones del sistema y el almacenamiento principal, ya que están optimizados para la memoria NAND Flash de los dispositivos móviles.

Sistema de Archivos Tipo de Uso Característica Clave
EXT4
(Extended File System 4) 		Particiones del sistema operativo y almacenamiento interno. Es el sistema de archivos histórico y por defecto de Linux. Es estable, robusto y se utiliza ampliamente en la mayoría de las particiones del sistema Android.
F2FS
(Flash-Friendly File System) 		Particiones de datos y almacenamiento interno. Es un sistema desarrollado por Samsung específicamente para memorias Flash. Ofrece mejor rendimiento y eficiencia energética que EXT4 en lectura y escritura aleatoria.
FAT32 / exFAT Tarjetas MicroSD (almacenamiento portátil). Se utilizan para garantizar compatibilidad con hardware y sistemas operativos externos como Windows y macOS al usar tarjetas de memoria.

3.5 Paginación en Android

La paginación y la segmentación en Android son mecanismos de gestión de memoria heredados del Kernel de Linux. Estos mecanismos son fundamentales para permitir la multitarea segura y optimizar el rendimiento de la memoria limitada en los dispositivos móviles.

La paginación es una técnica de gestión de memoria virtual que Android utiliza para asignar memoria física de manera eficiente y segura a cada aplicación.

Función principal

Permite que un proceso, es decir, una aplicación, se ejecute aunque su memoria no esté contigua en la RAM o aunque solo una parte de ella esté cargada. Esto se logra mediante el uso de memoria virtual.

Mecanismo

La memoria de un proceso, también llamada espacio de direcciones virtual, se divide en bloques de tamaño fijo llamados páginas, normalmente de 4 KB.

La memoria física o RAM también se divide en bloques del mismo tamaño, llamados marcos de página o page frames.

El sistema operativo, a través del Kernel de Linux, mantiene una tabla de páginas para cada proceso. Esta tabla relaciona las páginas virtuales con los marcos físicos en la RAM.

Beneficios en Android

Aislamiento de procesos (Seguridad)

Garantiza que una aplicación no pueda acceder a la memoria física de otra aplicación ni a la del sistema, reforzando así el modelo de seguridad o sandbox de Android.

Multitarea eficiente

Permite que el sistema use espacio en disco, aunque Android lo utiliza de forma limitada, para mover temporalmente páginas no utilizadas. Esto libera RAM para las aplicaciones activas y mejora la multitarea.

3.6 Segmentación en Android

La segmentación es un método de gestión de memoria donde el espacio de direcciones de un proceso se divide en segmentos lógicos de tamaño variable, como el código, los datos y la pila.

Uso en Android

En sistemas modernos como Linux, que es la base de Android, la segmentación no se utiliza como el mecanismo principal de gestión de memoria.

Android se basa casi completamente en la paginación para garantizar la protección y el aislamiento de la memoria.

Funcionamiento en Linux

Aunque la arquitectura x86 soporta segmentación mediante registros especiales, Linux configura un espacio de direcciones plano, donde todos los procesos comienzan desde la misma dirección base.

Esto simplifica la gestión de memoria y delega toda la complejidad a la paginación, que es más eficiente y segura.

Segmentación lógica en Android

Aunque no se usa directamente a nivel del sistema, las aplicaciones sí manejan una organización lógica de la memoria en diferentes áreas:

Código

Contiene las instrucciones de la aplicación.


Datos

Almacena variables y estructuras necesarias para el funcionamiento de la app.


Heap

Área de memoria dinámica donde se crean objetos durante la ejecución.


Stack

Se utiliza para manejar llamadas a funciones y ejecución del programa.

3.7 Asignación de memoria entre procesos

En Android, cada aplicación se ejecuta dentro de un proceso independiente, normalmente con una instancia del Android Runtime (ART). Esto permite que cada proceso tenga su propio espacio de memoria aislado.

Aislamiento de procesos

Cada proceso tiene su propio espacio de direcciones virtuales y no puede acceder directamente a la memoria de otras aplicaciones.

Android utiliza un modelo de sandbox para aislar procesos, mejorando la seguridad y estabilidad del sistema.

Asignación de memoria

El Kernel de Linux asigna a cada proceso un rango de direcciones de memoria virtual.

La MMU (Memory Management Unit) se encarga de traducir direcciones virtuales a direcciones físicas en la RAM.

La memoria se gestiona mediante paginación, utilizando tablas de páginas para controlar qué partes de la RAM están asignadas.

Estructura de memoria de un proceso

Heap

Área de memoria dinámica donde se crean objetos (Java/Kotlin).

Stack

Se utiliza para variables locales y ejecución de funciones.

Código

Contiene las instrucciones ejecutables de la aplicación.

Datos estáticos

Variables globales o estáticas de la aplicación.

Bibliotecas compartidas

Se cargan dinámicamente y pueden compartirse entre procesos.

Gestión con ART y Garbage Collector

Android utiliza ART para ejecutar aplicaciones y gestionar la memoria del heap.

El Garbage Collector (GC) libera automáticamente memoria de objetos que ya no se utilizan.

Android 14 utiliza un GC concurrente y generacional, lo que permite recolectar memoria sin detener completamente la aplicación y optimizar el rendimiento.

Proceso Zygote

Android no crea procesos desde cero. Utiliza un proceso base llamado Zygote, que ya contiene librerías y clases del sistema precargadas.

Cuando se inicia una app, el sistema clona Zygote (fork), lo que permite iniciar aplicaciones más rápido.

Se utiliza Copy-On-Write (COW), lo que permite compartir memoria entre procesos y solo duplicarla cuando es necesario.

Gestión de memoria y prioridad

Android controla el uso de memoria mediante el Low Memory Killer Daemon (LMKD).

Cuando la memoria es insuficiente, el sistema elimina procesos de menor prioridad para liberar recursos.

Esto evita bloqueos y mantiene la fluidez del sistema.
